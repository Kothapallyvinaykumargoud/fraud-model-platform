## 2026-07-21

- **Change signal**: Learner requested using real, "bank-like" fraud transaction data and referenced an uploaded enterprise-MLOps case-study document (Accenture-style writeup) as a pattern to draw from. That document was flagged to the learner as ChatGPT-generated case-study copy (retained web-search citation artifacts, unverifiable "$20 billion" stat) — treated as a pattern reference (feature store, explainability, drift-triggered retraining, governance), not a literal spec.

- **Scope added**: FR-17 (Real Feature Dataset), FR-18 (Lightweight Feature Store), FR-19 (Feature Attribution) under new §4.8/§4.9. §5 Non-Goals updated: the blanket "no real transaction data" exclusion narrowed to "no live bank integration" (real sampled data is now in scope); EKS/multi-AZ/Jenkins/Airflow/Datadog-ELK/IAM-KMS-VPC exclusions reaffirmed explicitly given the new reference document raised them. §6.1 in-scope range extended to FR-19. §8/§9 updated with the Kaggle competition-rules-acceptance caveat (Open Question 6) and the sample-size assumption.
- **Triage**: No phase-blockers — implementation proceeded directly (code-first, per learner's explicit direction) rather than routing through bmad-create-architecture/bmad-create-epics-and-stories first, consistent with how this project has operated since its original PRD (architecture/epics were never formally produced as separate artifacts either).
